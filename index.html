Code Review

Why we do code reviews

A code review, after a developer completes the coding process, is an essential step to the development of software. It allows a second viewpoint on the implementation and solution before the branch can be merged with an upstream branch such as a feature or main branch. Reviewers can act as second steps in finding bugs, logic issues, edge cases or other issues.

Developing a strong code review process sets a foundation for continuous improvement and prevents unstable code from shipping to customers. Code reviews should become part of a software development teamâs process to improve code quality and provide transparency to the codebase. The code review process is also an important part in spreading knowledge throughout an organization.

Enhancement Two - Algorithms and Data Structures

The embedded thermostat system I chose for enhancement was developed for the TI SimpleLink WiFi CC3220S LaunchXL platform. This system can read ambient temperatures, adjust a set-point target, and control heating elements based on changes in temperature. This system integrates hardware and software components to demonstrate proficiency in embedded systems programming. This artifact was originally created for my CS 350 coursework which focused on embedded system and programmable logic. This artifact manages user-defined set-points for temperature, monitors ambient temperature in real-time, and controls a heating element. Initially, the artifact used a loop-based scheduling mechanism for managing multiple tasks, such as temperature reading, button handling, and heating control. This approach, while functional, lacked efficiency and scalability, making it an ideal candidate for enhancements in the algorithms and data structures category.

This artifact has been selected to be included in my ePortfolio as it demonstrates my ability to implement and optimize algorithms in a real embedded system. This artifact can be enhanced to showcase my key skills, such as task planning, algorithm design and efficiency optimization. These are all critical in solving complex computing problems.

Several improvements have been made. A priority-based scheduler was introduced instead of one that used polling of a timer. The importance of tasks like temperature checks, button handling and display updates is now based on time. A priority queue was implemented to manage tasks efficiently. To ensure tasks are completed in order, they are given a priority and a deadline. A priority queue also eliminated redundant loops and checks, reducing the computational load of the system. The data structure was added to store the readings for display later or updating a database.

Aligned Outcomes:
  • Outcome 3 - the priority-based scheduling algorithm demonstrates my ability to design and evaluate an efficient computing solution, leveraging algorithmic principles to address task management challenges in an embedded system.
  • Outcome 4 - by integrating innovative techniques such as dynamic scheduling and priority queues, the enhancement showcases my ability to use well-founded techniques and tools to implement solutions that deliver value.
